Запуск программы через ./tracer _имя программы_

```
tracer_run()
│
├─ fork()
│   ├─ [дочерний] PTRACE_TRACEME → execvp → СТОП (ждёт трассировщика)
│   └─ [родитель] waitpid → проснулся, дочерний остановлен после exec
│
├─ настраиваем опции ptrace (следить за fork/clone/syscall)
├─ PTRACE_SYSCALL → "продолжай, но остановись на каждом syscall"
│
└─ цикл:
    waitpid(-1) → кто-то из tracee остановился
        │
        ├─ завершился?           → убрать из таблицы
        ├─ сделал fork/clone?    → добавить нового потомка в таблицу
        ├─ syscall-стоп?         → смотрим регистры (номер syscall,аргументы)
        └─ реальный сигнал?      → передать программе
```

- `process.c & process.h` определяют структуру и массив процессов, а также операции добавления, получения, удаления процесса из массива. Поиск процесса происходит по `pid`;
- `syscalls.c & syscalls.h` определяют массив системных вызовов, с которым работает программа. А также функции для работы с массивом: получение имени системного вызова и печать в stdout PID, имя syscall, аргументы, возвращаемое значение.
- `tracer.c & tracer.h` трейсит переданный процесс. 
    - Программа получает один процесс по `argv[1]`, этот процесс может порождать другие процессы. Суть трейсинга - отследить все дерево процессов.
    - Массив `pstate` хранит структуру каждого процесса: `pid` и `entering`. `entering` необходим, чтобы отслеживать вход и выход `syscall`.
    - `peek_string()` используется для демонстрации, позволяет прочитать аргумент процесса, который лежит в памяти трассируемого процесса. `PTRACE_PEEKDATA` читает память чужого процесса, но только по 8 байт. Строка может быть длиннее, поэтому исп. цикл.
    - В `tracer_run()` делаем `fork()` процесса и заменяем процесс с `execvp()`. Останавливаем выполнение с помощью `waitpid()`. После подмены `PTRACE_SYSCALL` позволяет установить опцию для остановки на каждом следующем `syscall`. По завершении процесс удаляется из массива `pstate`.
        - При появлении нового дочернего процесса, когда родитель сделал `fork/clone` — ядро уведомиляет через `PTRACE_GETEVENTMSG`, узнаём `PID`, регистрируем процесс и вешаем на него ptrace-опции для слежки.
        - `sig == (SIGTRAP | 0x80)` сигнал об остановке на границе `syscall`. В этот момент программа читает все регистры процессора для получение информации о системном вызове.
        `rax` на входе содержит номер `syscall`, а не результат, поэтому его читаем только на выходе `entering == 0`
        - `PTRACE_SYSCALL` отпускает процесс и остановит его на следующем `syscall`.
        - Когда все дочерние процессы завершаться, `waitpid` вернет -1 с `errno = ECHILD`. Цикл завершиться. 
- `eventbuf.c`кольцевой буфер между двумя потоками.
- `gui.c` GUI GTK4
